# 電子宠物 (Denshi Pet) - 设计规范

## 1. 项目概述

- **项目名称**: 電子宠物 (Denshi Pet)
- **类型**: 复古像素养成游戏
- **核心功能**: 喂养、玩耍、清洁、教育你的虚拟宠物
- **目标用户**: 怀旧玩家，90年代电子宠物爱好者
- **技术栈**: 单 HTML 文件，原生 JS + Canvas

## 2. 视觉风格

### 复古 LCD 风格
- **主色调**: 
  - 背景: 深灰 (#2d2d2d)
  - LCD 背景: 浅绿灰 (#9bbc0f) - 经典 Game Boy 色调
  - LCD 像素: 深绿 (#0f380f)
  - 外壳: 红色渐变 (#c0392b → #e74c3c)

### 像素艺术
- 宠物分辨率: 16x16 像素 (Canvas 绘制)
- 无抗锯齿，保持复古感
- 三种独特宠物，各有多种表情和动画

### 三种宠物
1. **🦞 龙虾 (LOBSTER)** - 红/橙色，横向摇摆 (waddle) 动画
2. **🐼 熊猫 (PANDA)** - 黑白像素，圆脸，高弹跳 (bounce) 动画
3. **🦭 海豹 (SEAL)** - 灰/白色，滑行 (slide) 动画

### 字体
- 主字体: "Press Start 2P" (Google Fonts) - 8-bit 风格

## 3. 游戏机制

### 宠物状态 (0-100)
| 状态 | 范围 | 影响 |
|-----|------|-----|
| 饥饿 | 0-100 | 低于 20 宠物不开心 |
| 开心 | 0-100 | 影响宠物表情 |
| 清洁 | 0-100 | 低于 20 会生病 |
| 纪律 | 0-100 | 影响宠物行为 |
| 年龄 | 0-3 | 0=蛋, 1=宝宝, 2=儿童, 3=成人 |

### 生命值 (HP)
- 初始: 100
- 饥饿<20: -1/秒
- 清洁<20: -1/秒
-开心<20: -0.5/秒
- HP=0: 宠物死亡

### 排泄系统
- 喂养后 30 秒产生便便
- 便便不清理: 开心 -5/10秒
- 最多 3 坨便便

### 生命周期
1. **蛋阶段** (0分钟) - 孵化动画，三阶段裂纹
2. **宝宝阶段** (0-5分钟) - 只能喂食、玩耍
3. **儿童阶段** (5-15分钟) - 解锁清洁、纪律
4. **成人阶段** (15+分钟) - 完全体

### 宠物选择系统
- 新游戏时随机选择三种宠物之一
- 首次进入显示宠物选择界面（可预览三种宠物像素画）
- 各宠物有独特 Idle 动画和专属表情

## 4. 交互设计

### 按钮布局 (模仿 Tamagotchi 4 按钮)
```
[选择] [设置]
  ↓     ↓
[A按钮] [B按钮]
```

### 按钮功能
- **选择键**: 循环切换功能图标
- **设置键**: 进入/确认
- **A按钮**: 减少/左
- **B按钮**: 增加/右

### 功能菜单
1. 🍖 **喂食** - 饥饿+30
2. 🎮 **玩耍** - 开心+25，但饥饿-10
3. 🛁 **清洁** - 清洁+40
4. 📚 **教育** - 纪律+20
5. 💡 **灯** - 开灯/关灯（睡觉用）

## 5. 宠物行为 AI

### 表情状态
- 😊 开心: 开心>60
- 😐 一般: 开心 30-60
- 😢 不开心: 开心<30
- 😠 生气: 纪律<20
- 🤒 生病: 清洁<20 或 饥饿<20

### 自动行为
- 饿了自己叫（屏幕闪烁）
- 便便满了自己叫
- 开心时自己玩（动画）
- 睡觉时闭眼呼吸动画

## 6. 音效设计

### 8-bit 复古音效
- 喂食: "滴-滴滴" (C-E-G)
- 玩耍: "滴滴滴滴" (快速)
- 清洁: "哗啦~" (滑音)
- 教育: "叮-叮-叮" (上课铃)
- 警告: "滴滴滴滴" (急促)
- 死亡: "~~~~~" (下坠音)

使用 Web Audio API 生成 8-bit 音效

## 7. 数据存储

- localStorage 保存宠物状态
- 关闭浏览器继续计时（离线惩罚）
- 可重置/新宠物
