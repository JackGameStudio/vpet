# Pixel Pet Design Notes

## Overview
This document describes the pixel art sprites designed for the Denshi Pet virtual pet game. Each pet has 4 states (neutral, happy, sad, sick/angry) plus a shared egg sprite with 3 crack stages.

## Canvas & Display
- **Sprite Size**: 16×16 pixels each
- **Display**: Centered on 400×300 canvas, scaled 5x with pixel-perfect rendering
- **Animation**: Gentle bobbing motion (2-3px amplitude)

## Sprite Data Format

Each sprite is stored as a 256-character string (16×16 grid). Each character maps to a color via the palette:

```javascript

  "r": "#CC2200",  // Lobster red
```

## State Selection Logic

The game selects sprites based on pet state:

```javascript
function sprKey() {
  if (phase === 0) { // Egg
    if (hatchProgress > 66) return "egg_crack3";
    if (hatchProgress > 33) return "egg_crack2";
    if (hatchProgress > 10) return "egg_crack1";
    return "egg_whole";
  }
  
  // Pet states
  if (dead) return petType + "_sad";
  if (lightsOff) return petType + "_sleep"; // optional
  if (hp < 20 || hunger < 20) return petType + "_sick";
  if (discipline < 20) return petType + "_sick"; // angry
  if (happiness < 30) return petType + "_sad";
  if (happiness > 60) return petType + "_happy";
  return petType + "_neutral";
}
```

## Design Principles

1. **Readability at Small Size**: All sprites use bold, simple shapes visible even at 16×16
2. **Distinct Silhouettes**: Each pet has a unique outline (lobster=long, panda=round, seal=sleek)
3. **Consistent Color Usage**: Same color always means same feature (e.g., black = panda limbs)
4. **Expressive States**: Emotional states are clearly distinguishable
5. **Game Boy Aesthetic**: Using limited palette reminiscent of original Game Boy games
